## 🏢 模块详细说明

### 1️⃣ 数据准备模块 (data_preparation.py)

**职责**: 数据加载、清洗和预处理

**核心类**: `DataPreparationModule`

**功能**:
- 📂 **文档加载** - 从 Markdown 文件加载剧情数据
- 🔍 **章节提取** - 提取章节和幕信息
- 📝 **元数据增强** - 添加角色、地点等元数据
- ✂️ **文档分块** - 按章节和幕进行分块
- 🗂️ **父子文档映射** - 建立文档层级关系

**关键方法**:
```python
load_documents()      # 加载所有 Markdown 文档
chunk_documents()     # 分块处理
extract_metadata()    # 提取元数据
```

---

### 2️⃣ 索引构建模块 (index_construction.py)

**职责**: 构建向量索引和关键词索引

**核心类**: `IndexConstructionModule`

**功能**:
- 📊 **向量索引** - 使用 FAISS 构建向量索引
- 🔑 **关键词索引** - 构建 BM25 关键词索引
- 💾 **索引保存** - 保存索引到磁盘
- 📥 **索引加载** - 加载已保存的索引

**关键方法**:
```python
build_vector_index()  # 构建 FAISS 向量索引
save_index()          # 保存索引
load_index()          # 加载索引
```

---

### 3️⃣ 检索优化模块 (retrieval_optimization.py)

**职责**: 混合检索和结果重排

**核心类**: `RetrievalOptimizationModule`

**功能**:
- 🔍 **向量检索** - 基于语义相似度的向量检索
- 🔑 **BM25 检索** - 基于关键词的检索
- 🎯 **混合检索** - 结合向量和关键词检索
- 🔄 **RRF 重排** - Reciprocal Rank Fusion 重排算法
- 🏷️ **元数据过滤** - 支持按角色、章节等过滤

**关键方法**:
```python
hybrid_search()           # 混合检索
_rrf_rerank()            # RRF 重排
metadata_filtered_search() # 元数据过滤检索
character_occurrence_search() # 角色出现检索
```

---

### 4️⃣ 生成集成模块 (generation_integration.py)

**职责**: LLM 集成和回答生成

**核心类**: `GenerationIntegrationModule`

**功能**:
- 🔄 **查询重写** - 优化用户查询以提高检索效果
- 🧭 **意图识别** - 分类问题类型（事实性/推理性/通用）
- 📝 **回答生成** - 基于检索结果生成回答
- 🔄 **重试机制** - 处理 API 限流

**关键方法**:
```python
rewrite_query()              # 查询重写
query_router()              # 意图识别
generate()                  # 回答生成
extract_structured_entities() # 结构化实体提取
```

---

### 5️⃣ Neo4j 图检索模块 (graph_neo4j_retrieval.py)

**职责**: 基于 Neo4j 图数据库的检索

**核心类**: `Neo4jGraphRetrieval`

**功能**:
- 🔗 **Neo4j 连接** - 连接到 Neo4j 图数据库
- 📋 **角色查询** - 查询角色详细信息
- 🤝 **关系查询** - 查询角色关系网络
- 📅 **出现查询** - 查询角色首次/最后出现
- 📍 **地点查询** - 查询角色出现地点
- 📊 **统计信息** - 获取图数据库统计信息

**关键方法**:
```python
query_character_info()           # 查询角色信息
query_character_relationships()  # 查询角色关系
query_character_first_appearance() # 首次出现
query_character_last_appearance()  # 最后出现
query_character_network()        # 完整关系网络
get_statistics()                 # 统计信息
```

---

### 7️⃣ RAG 系统核心 (main.py)

**职责**: 整合所有模块，提供统一的问答接口

**核心类**: `RAGSystem`

**功能**:
- 🔧 **系统初始化** - 初始化所有模块
- 📚 **知识库构建** - 构建/加载知识库
- 🤖 **问答处理** - 处理用户问题
- 🎯 **智能路由** - 根据问题类型选择查询方式
- 📝 **结果生成** - 生成最终回答

**关键方法**:
```python
initialize_system()           # 初始化系统
build_knowledge_base()        # 构建知识库
ask_question()               # 问答处理
_query_character_relationships_neo4j() # Neo4j 关系查询
_query_character_occurrence_neo4j()    # Neo4j 出现查询
```

---

### 7️⃣ 数据提取模块 (graph_data_extractor.py)

**职责**: 使用 LLM 从剧情文档提取图数据

**核心类**: `GraphDataExtractor`

**功能**:
- 🤖 **LLM 提取** - 使用大模型提取实体和关系
- 📝 **结构化输出** - 生成 nodes.csv 和 relationships.csv
- 🗃️ **数据导出** - 导出为 CSV 格式
- 📜 **Cypher 生成** - 生成 Neo4j 导入脚本

**关键方法**:
```python
extract_from_documents()  # 从文档提取图数据
generate_csv()           # 生成 CSV 文件
generate_cypher()        # 生成 Cypher 脚本
```

## 🔄 数据流程

### 1. 数据准备流程

```
Markdown 文件
    ↓
[DataPreparationModule]
    ↓
加载文档 → 提取元数据 → 分块处理
    ↓
Document 列表 (带元数据)
```

### 2. 索引构建流程

```
Document 列表
    ↓
[IndexConstructionModule]
    ↓
向量嵌入 → FAISS 索引 → 保存索引
    ↓
向量索引文件
```

### 3. 图数据提取流程

```
Document 列表
    ↓
[GraphDataExtractor]
    ↓
LLM 提取 → 结构化数据 → CSV 文件
    ↓
Neo4j 图数据库
```

### 4. Neo4j 导入流程

```
CSV 文件 (nodes.csv, relationships.csv)
    ↓
[setup_neo4j_import.py]
    ↓
复制文件 → Neo4j 导入 → 验证数据
    ↓
Neo4j 图数据库
```

### 5. 问答流程

```
用户问题
    ↓
[RAGSystem.ask_question()]
    ↓
┌────────────────────────────────────────────────────────────┐
│  查询重写 (LLM)                                            │
│  → 优化查询以提高检索效果                                   │
└────────────────────────────────────────────────────────────┘
    ↓
┌────────────────────────────────────────────────────────────┐
│  意图识别 (LLM)                                            │
│  → 分类: factual / inferential / general                  │
└────────────────────────────────────────────────────────────┘
    ↓
┌────────────────────────────────────────────────────────────┐
│  结构化实体提取 (LLM)                                       │
│  → 提取: character, location, chapter, action, time       │
└────────────────────────────────────────────────────────────┘
    ↓
┌────────────────────────────────────────────────────────────┐
│  查询路由 (根据实体和意图)                                  │
└────────────────────────────────────────────────────────────┘
    ↓
    ├─→ [Neo4j 查询] (如果有 Neo4j 且有角色实体)
    │   └─→ 关系网络查询 / 出现查询
    │
    └─→ [混合检索] (常规检索)
        ├─→ 向量检索 (FAISS)
        ├─→ BM25 检索
        └─→ RRF 重排
    ↓
┌────────────────────────────────────────────────────────────┐
│  上下文格式化                                               │
│  → 整理检索结果为上下文                                     │
└────────────────────────────────────────────────────────────┘
    ↓
┌────────────────────────────────────────────────────────────┐
│  LLM 生成回答                                               │
│  → 基于上下文生成最终回答                                   │
└────────────────────────────────────────────────────────────┘
    ↓
用户回答
```

### 关系类型

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| APPEARS_IN | 角色出现在某章节 | 漂泊者 → 第0章 |
| LOCATED_AT | 角色位于某地点 | 漂泊者 → 今州 |
| INTERACTS_WITH | 角色互动 | 漂泊者 ↔ 秧秧 |
| ENEMY | 敌人关系 | 漂泊者 → 伤痕 |
| ALLY | 盟友关系 | 秧秧 → 炽霞 |
| OWNS | 拥有物品 | 漂泊者 → 葫芦 |
| MEMBER_OF | 所属组织 | 漂泊者 → 黑海岸 |
| FAMILY | 家庭关系 | 角色A ↔ 角色B |
| RELATES_TO | 相关关系 | 角色A → 事件 |

## 🎯 查询类型

### 1. 事实性查询
**特点**: 需要精确的事实信息
**示例**:
- "漂泊者第一次出现在哪里？"
- "秧秧最后一次出现在第几章？"
- "炽霞和谁战斗过？"

**处理流程**:
```
用户问题 → 查询重写 → 意图识别(factual) → Neo4j/本地图查询 → 事实回答
```

### 2. 推理性查询
**特点**: 需要分析和推理
**示例**:
- "漂泊者和谁是一对？"
- "漂泊者最恨谁？"
- "谁对漂泊者可能有好感？"

**处理流程**:
```
用户问题 → 查询重写 → 意图识别(inferential) → Neo4j/本地图查询 → 推理回答
```

### 3. 通用查询
**特点**: 不涉及具体剧情
**示例**:
- "你是谁？"
- "你有什么功能？"

**处理流程**:
```
用户问题 → 意图识别(general) → 通用回答
```
